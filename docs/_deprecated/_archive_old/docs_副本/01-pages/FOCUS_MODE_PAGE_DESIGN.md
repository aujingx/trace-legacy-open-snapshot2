# Focus Mode 专注模式 - 页面设计文档

## 📋 概述
专注模式的核心目标不是「帮你计时」，而是**帮你跨过「开始专注」的那道心理门槛**。

> 大多数人不是不能专注 25 分钟，而是「决定要开始专注 25 分钟」这件事本身太有压力了。

## 💡 设计优化讨论点

### 1. 启动门槛 - 最难的是「决定开始」

**当前设计问题**：
用户进入专注模式页面，看到的是「25分钟」计时器 + 「开始」按钮。
心理活动：「25分钟？我现在能不能集中那么久？万一中间被打断怎么办？还是算了吧...」

**可以讨论的优化方向**：

#### 选项 A：「先做 5 分钟」模式 - 最低门槛
```
今天有点难以开始？ 👇

[ 🐣 先做 5 分钟试试 ]

如果 5 分钟后不想继续，随时可以停。
没有任何压力。
```

**设计哲学**：不是承诺 25 分钟，是承诺「只做 5 分钟」。
大多数人开始了就会继续，但「开始」是最难的。

#### 选项 B：「不限时」专注 - 去掉时间压力
```
[ 🌊 进入心流模式 ]

不设置计时器，
想专注多久就专注多久，
想停的时候随时停。
```

**设计哲学**：对于创造性工作，番茄时间反而是打断。
有时候进入状态了，25 分钟响铃反而是破坏心流。

#### 选项 C：多选项并列
```
[ 🐣 5分钟试试 ]  ·  [ 🍅 25分钟标准 ]  ·  [ 🌊 不限时 ]
```

**需要讨论的问题**：
- 你自己在难以启动的时候，什么样的设计最能说服你「先开始再说」？
- 「5分钟模式」结束的时候应该怎么反馈？是直接结束，还是问「要不要继续」？
- 「不限时模式」是不是应该作为高级选项？还是默认就给用户？

---

### 2. 专注过程中的体验

**当前设计问题**：只有一个倒计时数字，用户盯着数字反而焦虑。

**可以讨论的优化方向**：

#### 概念：「过程奖励」而不是「倒计时压力」
```
✅ 你已经专注了 17 分钟

这 17 分钟里，你没有切换到社交软件
很棒，继续保持 🌟
```

而不是：
```
⏰ 还有 8 分钟
```

#### 概念：温柔的中途提醒
如果检测到用户频繁切换应用（分心了），不是弹警告：
```
检测到你最近有点分心
要不要站起来喝口水，重新集中一下？ 🚰
```

**需要讨论的问题**：
- 专注过程中，你希望看到什么信息？还是什么都不看最好？
- 分心中途的提醒，会不会反而造成更大的打扰？
- 提醒的语气应该是什么样的？严厉？温柔？中立？

---

### 3. 专注结束后的正反馈

**当前设计问题**：时间到了就响铃，然后进入休息。没有成就感。

**可以讨论的优化方向**：

#### 概念：「专注总结卡片」
每次专注结束，展示：
```
🎉 专注完成！

这次你专注了 32 分钟
期间只切换了 2 次应用
比你平均水平高 15%

🏅 这是你今天第 3 次成功专注
```

#### 概念：「渐进式休息建议」
不是固定 5 分钟休息：
```
这次你专注了 32 分钟
建议休息 3-5 分钟

站起来走走，看看远处，
不要刷手机。

[ 开始休息 ]
```

**需要讨论的问题**：
- 每次专注结束都显示总结会不会太频繁？还是只在特别长的专注后显示？
- 总结应该包含哪些信息才让人觉得有成就感？
- 休息结束的提醒应该是什么样的？

---

### 4. 「启动倦怠」的特殊处理

这是你今天提到的核心问题：想到要做事就麻木，完全不想启动。

**可以讨论的设计方向**：

#### 概念：「我今天不想做事」模式
用户可以主动标记：
```
今天状态不好，我承认我不想做事。

没关系，不强迫自己。
我们就先做「一件最小的事」。

👇 选一个你能接受的最小目标：

[ 2 分钟 ]  ·  [ 5 分钟 ]  ·  [ 10 分钟 ]

做完就收工，绝不要求更多。
```

**设计哲学**：不要和倦怠对抗，承认它，然后绕开它。
目标不是「高效工作」，目标是「不要打破连续工作的 streak」。

**需要讨论的问题**：
- 你觉得这种「认输」式的设计有用吗？
- 会不会反而让用户觉得「这个软件允许我偷懒」？
- 还是说，这反而能降低心理负担，让用户更容易开始？

## 🎨 参考案例

### Forest 专注森林
- 优点：正反馈强（种树），社交压力激励
- 缺点：太游戏化了，工作的时候想着树反而分心

### Cold Turkey
- 优点：屏蔽力度强，真的能让人专注
- 缺点：太强硬，像家长管孩子，有逆反心理

### Endel
- 优点：用声音创造专注氛围，没有压力
- 缺点：没有追踪和反馈
