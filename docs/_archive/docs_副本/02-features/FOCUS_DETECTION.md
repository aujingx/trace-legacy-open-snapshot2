# 🔍 全局智能分心检测系统

> **设计原则**: 像一个温柔的助手，而不是一个严厉的老师。帮助用户觉察，而不是审判。

---

## 🎯 核心问题场景

| 场景 | 说明 |
|------|------|
| **用户去刷短视频/看剧了** | 活动分类属于「娱乐」，持续 > 3 分钟 |
| **用户和朋友闲聊了** | 微信/QQ 等 IM 应用，持续 > 5 分钟 |
| **用户去开会了** | Zoom/Teams/会议软件，可能是正常工作 |
| **用户去查资料了** | 浏览器但访问的是技术网站 |
| **用户只是盯着屏幕思考** | 没有鼠标键盘输入，但窗口还是 IDE |

**难点**: 同一个行为对不同人意义完全不同
- 看剧 = 摸鱼 / 看剧 = 产品经理做竞品分析
- 聊天 = 摸鱼 / 聊天 = 销售在和客户沟通

## 🧠 分层检测逻辑

```
┌─────────────────────────────────────────────────────────────┐
│                     第一层：基础状态检测                        │
├─────────────────────────────────────────────────────────────┤
│  ✅ 键盘鼠标空闲时间 > 阈值（默认 5 分钟）                     │
│  ✅ 前台应用切换（从工作类 → 非工作类）                        │
│  ✅ Trace 窗口最小化/失焦                                     │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                     第二层：活动分类判断                        │
├─────────────────────────────────────────────────────────────┤
│  当前活动分类 == 「分心拦截列表」中的分类 → 触发弹窗          │
│  当前活动分类 == 「工作」→ 不触发                             │
│  当前活动分类 == 「会议/学习」→ 延长阈值（15 分钟）           │
│  当前活动分类 == 「未知」→ 结合历史习惯判断                   │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                     第三层：用户历史反馈                        │
├─────────────────────────────────────────────────────────────┤
│  过去用户对同类活动标记过「以后都不是」→ 跳过本次检测         │
│  过去用户标记过「这次先休息」→ 当天同类活动不再弹窗           │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                     第四层：连续专注正反馈                      │
├─────────────────────────────────────────────────────────────┤
│  连续专注超过 30 分钟 → 发送休息提醒                          │
│  之后每隔 30 分钟提醒一次「要不要休息一下？」                 │
│  用户选择休息 → 重置专注计时                                  │
│  用户继续工作 → 专注计时延续，继续按间隔提醒                   │
└─────────────────────────────────────────────────────────────┘
```

## 🔄 用户反馈后的后续动作

| 用户选择 | 系统行为 |
|---------|---------|
| **✅ 是的，我在工作** | ① 这段时间标记为工作<br>② 恢复连续专注时长计时<br>③ 24 小时内不检测微信 |
| **🤷 这次先休息** | ① 这段时间标记为「休息」<br>② 开始 15 分钟「休息计时」，期间不弹窗打扰<br>③ 重置连续专注时长<br>④ 休息结束后温柔提醒「该回来工作啦 👋」 |
| **❌ 以后都不是** | ① 这段时间标记为「分心」<br>② 重置连续专注时长<br>③ 把这个应用/网站永久加入「分心拦截列表」<br>④ 下次检测到直接自动拦截并记录 |
| **⚙️ 自定义规则** | 弹出规则配置页，用户可以自定义：<br>• 这个应用属于什么分类？<br>• 超过多长时间才提醒？ |

## ⚙️ 可配置阈值

```typescript
interface DetectionSettings {
  idleThresholdMinutes: number      // 空闲超时提醒：默认 5 分钟
  distractionThresholdMinutes: number  // 分心应用超时：默认 3 分钟
  meetingThresholdMinutes: number   // 会议类超时：默认 15 分钟
  breakAutoReturnMinutes: number    // 休息自动提醒：默认 15 分钟
  enableNotification: boolean       // 最小化时用系统通知：默认 true
  enableSound: boolean              // 提醒声音：默认 false
  focusReminderThresholdMinutes: number  // 连续专注多久后开始提醒：默认 30 分钟
  focusReminderIntervalMinutes: number   // 每隔多久提醒一次：默认 30 分钟
}
```

## 🎯 连续专注时长统计

**核心设计原则**：连续专注时长 = 从进入工作状态开始，到休息/分心/长时间空闲为止的总时长，中间切换任务不中断计时。

### 计时逻辑

```
用户开始工作（VS Code）
    ↓ focusStartTime = now
专注 15 分钟
    ↓ 切换到 Figma（仍然是工作分类）→ 计时不中断
继续专注 20 分钟
    ↓ 总时长 = 35 分钟，达到提醒阈值 → 发送休息提醒
    ↓ 每隔 30 分钟提醒一次（可配置）
用户选择休息
    ↓ focusStartTime = null，计时重置
休息 10 分钟
    ↓ 回到工作状态 → 重新开始计时
```

### 中断专注计时的场景

| 场景 | 行为 |
|------|------|
| **开始休息** | ✅ 重置专注计时 |
| **用户标记为分心** | ✅ 重置专注计时 |
| **空闲超时（默认 5 分钟）** | ✅ 重置专注计时 |
| **分心应用超时（默认 3 分钟）** | ✅ 重置专注计时 |
| **从 VS Code 切换到 Chrome（都是工作）** | ❌ 不中断，计时延续 |
| **从 工作A 切换到 工作B** | ❌ 不中断，计时延续 |

### 公开 API

```typescript
// 获取当前连续专注时长（分钟）
const minutes = useFocusStore.getState().getContinuousFocusMinutes()

// 状态订阅
const focusMinutes = useFocusStore(state => state.getContinuousFocusMinutes())
```

## 📊 数据记录与统计

每次弹窗和用户选择都会被记录，用于后续分析用户习惯：

```typescript
interface DistractionRecord {
  timestamp: string
  appName: string
  windowTitle: string
  durationMinutes: number
  userChoice: 'work' | 'break' | 'distraction' | 'dismissed'
  autoDismissed: boolean  // 用户没点，超时自动关闭
}
```

**后续可以做的分析**：
- 一天被打断多少次？
- 最容易分心的应用是什么？
- 平均每次分心持续多久？
- 什么时间段专注力最好？

## 🏗️ 技术实现架构

```
src/
├── services/
│   └── focusDetection.ts      # 核心检测逻辑（单例）
├── store/
│   └── useFocusStore.ts       # 全局状态管理
├── components/
│   └── DistractionDialog.tsx  # 分心弹窗组件
└── hooks/
    └── useFocusDetection.ts   # React 层封装
```

**核心特点**：
1. **全局单例**：检测逻辑独立运行，不绑定任何页面
2. **事件驱动**：空闲检测器触发事件 → 检测逻辑处理 → 派发状态更新
3. **可中断**：用户回到工作窗口后可以随时取消弹窗
4. **幂等性**：同一次分心活动不会重复弹窗
