# MRD - Trace 时迹
## 市场需求文档

**文档版本**: v1.0  
**创建日期**: 2026-04-28  
**产品名称**: Trace 时迹 - AI自动时间追踪工具  
**目标用户**: 知识工作者、开发者、自由职业者、远程工作团队

## 1. 问题定义

### 1.1 用户痛点深度分析

| 用户角色 | 典型场景 | 核心痛点 | 现有解决方案 | 不满之处 |
|----------|----------|----------|--------------|----------|
| **独立开发者小陈** | 在家接外包，需要给客户报工时 | 1. 经常忘记开计时器，漏掉大量时间<br>2. 手动记录太麻烦，坚持不下来<br>3. 国外工具全英文，看着累，访问还慢 | Toggl + 提醒事项 | 需要手动点开始/停止，经常忘 |
| **远程工作的产品经理小林** | 团队分散各地，经理想知道大家工作状态 | 1. 日报周报太形式主义，浪费时间<br>2. 不知道大家每天真正有效工作时长<br>3. 国外团队工具国内用不了 | 飞书任务 + 口头同步 | 数据太粗，没有真实时间维度 |
| **考研二战的小张** | 在家备考，经常玩手机分心 | 1. 想知道每天真正学习了多久<br>2. 需要专注模式屏蔽娱乐网站<br>3. 希望有热力图激励自己坚持 | Forest 种树 + 番茄钟 | 只能计时，不知道具体时间花在哪了 |
| **自由设计师阿美** | 同时接好几个项目，月底对账才发现亏了 | 1. 不知道每个项目具体花了多少小时<br>2. 经常因为估时不准报价亏了<br>3. 没有数据证明自己确实工作了那么久 | Excel 手工记录 | 太麻烦，经常漏记，统计分析困难 |

### 1.2 共性痛点总结
1. **手动记录反人性**：没有正常人能坚持每次工作都点开计时器
2. **时间黑洞**："我今天都干啥了？感觉忙了一天但啥也没干成"
3. **估时不准**：对"完成一项任务需要多久"永远估计错误
4. **分心严重**：刷微信、刷短视频不知不觉就浪费了几小时
5. **国外工具水土不服**：英文界面、访问慢、价格贵、没有国内软件集成

### 3.2 用户体验要求

#### 设计原则（Aether 通透现代设计系统）
1. **通透背景**：浅灰 `#f0f0f0` + 干净白色卡片，呼吸感强
2. **超大圆角**：容器28px / 卡片16px / 按钮10px，现代风格
3. **柔和阴影分层**：无边框设计，仅用阴影区分层级
4. **五种主题配色**：天蓝（默认）、翠绿、紫调、橙黄、樱花粉，用户可自由切换

#### 交互原则
1. **静默优先**：90%的功能后台自动完成，不需要用户操作
2. **一键操作**：常用功能（开始专注、暂停追踪）一键搞定
3. **全局快捷键**：不打开窗口也能快速操作
4. **系统托盘**：状态栏常驻入口，快速查看状态
5. **新手导览**：交互式引导，新用户5分钟上手

#### 性能要求
- 安装包大小 < 50MB
- 内存占用 < 100MB（后台静默）
- CPU占用 < 1%（空闲时）
- 启动时间 < 2秒
- 电池影响 < 5%/天（MacBook）

### 4.2 我们的独特优势

1. **真正的一站式**：自动追踪 + 任务管理 + 番茄工作法 + 专注模式，不用装好几个App
   - 竞品都是单一功能，用户需要在Rize + Toggl + 番茄钟之间切换

2. **中文深度优化**：不只是翻译，针对国内软件生态专门优化
   - 微信、飞书、钉钉、WPS、VS Code中文等国内常用应用分类更准
   - 手机号/微信登录，符合国内用户习惯

3. **Aether设计系统**：颜值就是正义
   - 参考顶尖Figma仪表盘设计，超大圆角 + 柔和阴影 + 通透背景
   - 五种主题配色自由切换，满足不同用户审美偏好

4. **Agent原生支持**：面向未来的架构
   - 所有核心能力通过标准化API开放
   - 未来AI Agent可以自动发现并调用，真正实现"AI帮你管理时间"
