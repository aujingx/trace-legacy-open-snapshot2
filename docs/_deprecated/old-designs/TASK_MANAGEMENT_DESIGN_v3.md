# 任务管理 - 页面设计文档 v3.0

> **最后更新**: 2026-05-02
> **设计哲学**: 我们不做「更好的 GTD」，我们做「让你愿意创建任务，并且愿意开始做任务」的任务管理。
>
> **核心矛盾**: 创建任务的门槛越高，用户就越不想创建任务；任务列表越长，用户越容易产生「我什么都不想做」的麻木感。
>
> **特邀顾问视角**:
> - 🗣️ 张一鸣：信息效率 + 用户体验 + 数据驱动产品迭代
> - 🚀 马斯克：第一性原理 + 极限创新 + 攻击本质问题

## ============================================================
## 第一性原理思考（马斯克视角）
## ============================================================

### 灵魂拷问：用户为什么需要任务管理软件？

> ❌ 错误答案：「为了管理更多任务，完成更多事情」
>
> ✅ 正确答案：「为了减少焦虑，知道自己现在应该做什么」

### 极限推论
如果用户打开软件只需要知道「我现在应该做什么」——

**敢不敢默认视图就只显示一个任务？**

```
                    🎯 现在你应该做这个：

              准备产品演示（预计 45 分钟）

      🐣 第一步：打开 Keynote，新建一个空白文档


      [ 开始专注 ]       [ 换一个推荐 ]       [ 我不想做这个 ]
```

> 其他任务？全部藏起来。用户想看的时候再看。90% 的时候，用户只需要知道「接下来做什么」。

## P0: 行业 UX 基线对齐（消除别扭感）
## ============================================================

### 1. 「一行输入」快速创建任务

#### 交互规则（行业标准）
- 用户只输入标题，按回车 → **立即创建成功**
- 其他所有字段都是**可选的，事后编辑**
- 支持自然语言智能解析

#### 张一鸣建议：AI 自动建议第一步
```
用户输入："写报告"
  ↓ AI 自动弹出 ↓

🐣 AI 建议你从这开始，选一个：
  ○ 先建个空白文档
  ○ 列个大纲要点
  ○ 找一下上次的报告参考
  ○ 我自己写第一步：[_______]
```

> **逻辑**：用户拖延的核心障碍不是任务本身，是「不知道从哪开始」。帮他把第一步决策做好，行动概率提升 500%。

### 2. 任务卡片 UX 重构

#### 单复选框原则
```
┌─────────────────────────────────────────────────────┐
│ ✓  准备产品演示                                  │
│    🏷️ 优先级高  ·  ⚠️ 今天截止                    │
│    🐣 第一步：打开 Keynote                         │
│    P1  😰  📅 今天  ⏱️ 45分钟                      │
│                                      [▶️] [😔]    │
└─────────────────────────────────────────────────────┘
```

#### 交互规则
| 操作 | 结果 |
|------|------|
| 点击左侧复选框 | 标记完成 / 取消完成 |
| 点击卡片其他区域 | 打开右侧详情面板 |
| 长按卡片 0.5 秒 | 进入多选模式 |
| 右滑卡片 | 显示「删除」「推迟到明天」 |

---

### 3. 三档视图密度（用户可选择）

| 视图模式 | 展示内容 | 适用场景 |
|---------|---------|---------|
| 🎯 **单任务模式** | 默认只显示 AI 推荐的「现在最应该做」的 1 个任务 | 焦虑感重、选择困难的用户 |
| 📋 **今日模式** | 只显示今日 + 逾期任务，其他折叠 | 大部分用户的默认选择 |
| 📂 **全部模式** | 显示所有任务 + 筛选 | 需要全局管理的时候 |

> **马斯克洞见**：不要用「更多功能」证明产品价值，要用「更少焦虑」证明价值。

## P1: 核心差异化功能（AI + 可视化）
## ============================================================

### 4. AI 智能拆解大目标

#### 拆解原则（升级后）
1. **第一个任务必须能在 15 分钟内完成**（降低启动门槛）
2. 每个任务有 AI 建议的「第一步」
3. 标注「能量等级」：🔴 高能量 / 🟡 中能量 / 🟢 低能量

#### 拆解结果示例
```
🎯 目标：两周内完成项目报告
进度：[░░░░░░░░░░] 0%

AI 帮你拆解成 5 个可执行的小任务：

1. [ ] 收集历史数据和参考资料 🟢 低能量
   🐣 第一步：打开文档库找上次的报告
   ⏱️ 预估 15 分钟  ·  P2  ·  现在就可以开始

2. [ ] 撰写大纲和章节结构 🟡 中能量
   🐣 第一步：在纸上画 3 个核心章节
   ⏱️ 预估 30 分钟  ·  P2  ·  完成第1步后开始

3. [ ] 填充各章节内容 🔴 高能量
   🐣 第一步：先写最简单的那一章
   ⏱️ 预估 120 分钟  ·  P1  ·  分 3 次完成

[ 💾 全部保存 ]  [ ✏️ 手动调整 ]  [ 🔄 AI 重新拆解 ]
```

### 5. 能量匹配推荐系统（马斯克首创）

#### 设计背景
> 「拖延」不是 bug，是人类的节能机制。用户不是不想做事，是当前的能量状态和任务需要的能量不匹配。

#### 能量等级定义
| 能量等级 | 任务类型 | 用户状态 |
|---------|---------|---------|
| 🔴 高能量 | 需要深度专注、创造性思考、困难决策 | 早上、休息好、精力充沛 |
| 🟡 中能量 | 需要思考但不难、常规工作、沟通协作 | 白天正常状态 |
| 🟢 低能量 | 机械性操作、整理、回复邮件、简单核对 | 晚上、累了、不想动脑 |

#### 智能推荐逻辑
```typescript
function recommendTask(tasks, userState) {
  // 1. 检测用户当前能量等级
  //    - 根据时间（晚上 = 低能量）
  //    - 根据历史完成模式（用户 22:00 从不做高能量任务）
  //    - 根据最近专注时长（连续专注 2 小时 = 能量下降）
  
  // 2. 优先推荐匹配能量等级的任务
  const energyMatched = tasks.filter(t => t.energyLevel === userState.currentEnergy)
  
  // 3. 其次推荐低一级的（宁肯做简单的，也不要不做）
  const lowerEnergy = tasks.filter(t => t.energyLevel < userState.currentEnergy)
  
  return sortByPriority([...energyMatched, ...lowerEnergy])
}
```

## P2: 情感连接与数据闭环
## ============================================================

### 7. 「我不想做」完整功能 + 数据采集

#### 用户侧体验
```
            我理解 😌
不想开始是很正常的，不用责怪自己。

👇 你可以试试：

┌─────────────────────────────────┐
│ ⏱️ 就做 5 分钟                  │
│    开启一个 5 分钟的专注，做完   │
│    就可以停下来，没有压力。      │
├─────────────────────────────────┤
│ 🐣 换个更简单的                  │
│    推荐一个 10 分钟就能做完的    │
│    低能量任务，先热个身。        │
├─────────────────────────────────┤
│ 📅 明天再说                      │
│    自动把截止日期改到明天，       │
│    今天就放过自己。              │
└─────────────────────────────────┘
```

#### 张一鸣建议：数据闭环设计
**不要只把「我不想做」当功能，把它当数据采集入口。**

每次用户点击「我不想做」，记录以下数据：
```typescript
interface ProcrastinationRecord {
  taskId: string
  taskType: string           // 工作/学习/生活
  taskEnergyLevel: 'high' | 'medium' | 'low'
  timestamp: Date            // 什么时间不想做
  timeOfDay: number          // 小时 0-23
  weekday: number            // 星期几
  userChoice: '5min' | 'easier' | 'tomorrow'  // 用户最后选了什么
  beforeFocusDuration: number  // 之前已经专注了多久
}
```

**积累 1000 条数据后，你就能预测：**
> 「用户每周五晚上 8 点以后，绝对不想做高能量的开发任务。下次别推荐了。」

### 8. 任务完成正反馈 + 心流中断保护

#### 完成庆祝弹窗
```
          🎉 太棒了！
「准备产品演示」完成了！
预估 45 分钟，实际花了 38 分钟，快了 7 分钟！

这是你今天完成的第 4 个任务。
你已经专注了 2 小时 13 分钟。


          [ 休息 5 分钟 ]  [ 继续下一个 ]
```

#### 张一鸣建议：心流中断保护
> 用户完成一个任务后，不要马上让他选下一个，而是给 30 秒的「成就时刻」。
>
> 字节的视频流是让你一直刷，但任务管理相反，要让用户有「完成一个阶段」的仪式感。

### 9. 「反目标」功能（马斯克首创）

#### 设计背景
> 所有任务管理软件都在让你「完成更多」，但很多时候，人们焦虑的来源是「给自己定的目标太多了」。

#### 功能设计
```
┌─────────────────────────────────────────┐
│ 🛡️ 本周保护计划                          │
│                                         │
│ 这周我只完成 __3__ 个任务就够了。        │
│                                         │
│ 系统会自动把其他任务全部推迟到下周。      │
│ 这一周，你只需要专注这 3 个。            │
│                                         │
│  [ ] 少即是多，允许自己做得少但做得好    │
│                                         │
│         [ 开启保护模式 ]                 │
└─────────────────────────────────────────┘
```

> **深层逻辑**：允许用户「做得少」反而能减少焦虑，提升完成质量。焦虑减少了，行动意愿反而会提升。

## 🎯 核心设计原则总结

### 张一鸣三大原则
1. **AI 不是炫技，是减负** —— AI 的价值不是展示你有多聪明，而是帮用户省脑子
2. **每一次交互都是数据** —— 用户的每一个选择都是偏好信号，积累起来就是护城河
3. **少即是多，慢即是快** —— 给用户的信息越少，用户越能行动；让用户有仪式感，长期留存越高

### 马斯克三大原则
1. **第一性原理思考** —— 不断问「用户真正需要的是什么」，不要被行业惯例限制
2. **利用人性，不要对抗人性** —— 拖延不是缺点，是人类的节能机制，顺着它设计
3. **反常识创新** —— 所有人都在做「更多」，你就做「更少」；所有人都在「催用户」，你就「保护用户」
