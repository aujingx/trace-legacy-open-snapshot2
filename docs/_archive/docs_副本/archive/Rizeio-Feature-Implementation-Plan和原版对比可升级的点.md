# Rize.io 功能分析与 Merize 实现计划

本文档基于 `_Rize.io产品分析文档.md` 整理，分析哪些功能值得 Merize 实现，按优先级排序。

---

## 概述

Rize.io 是一款**隐私优先的自动时间追踪工具**，定位 "Time tracking that runs itself"，核心目标是帮团队把时间自动转化为盈利、生产力和工作量洞察，35万+专业用户，主要服务代理团队、营销团队、专业服务团队。

Merize 目前已经实现了：
- ✅ 自动时间追踪（Tauri 原生）
- ✅ AI 自动分类（火山引擎）
- ✅ 时间块规划
- ✅ 专注模式 + 双层休息提醒
- ✅ 虚拟宠物 + 游戏化（我们**独有**差异化卖点）
- ✅ 团队 HR 游戏化框架
- ✅ 中文本地化 + 国产 AI 支持

本文档分析 Rize 哪些功能值得我们添加。

---

## 一、高优先级 - 现在就能做，产品价值大

这些功能添加后，能让 Merize 从"个人效率工具"升级到"专业生产力工具"，覆盖自由职业者和小团队的计费需求。

| 功能 | 价值说明 | 当前 Merize 状态 | 开发工作量 |
|------|---------|-----------------|------------|
| **三级分类体系 (客户 → 项目 → 任务)** | 满足自由职业者/代理团队按客户/项目计费的需求 | 只有任务层，缺少客户/项目层级 | ⭐⭐⭐ 中等 |
| **可计费/非可计费标记 + 费率设置** | 准确计算每个客户/项目的收入、成本、利润，为开票做准备 | 无 | ⭐⭐ 小 |
| **自定义追踪规则** | 用户可以添加关键词，教 AI 识别特定工作，大幅提升自动分类准确率 | 只有全局 AI 分类，缺少用户自定义规则 | ⭐⭐ 小 |
| **分心拦截器** | 可自定义规则，拦截分心的应用和网站，保证专注会话不被打断 | 专注模式已有基础，容易扩展 | ⭐⭐ 小 |
| **Focus Quality Score 专注质量分数** | 整合 20+ 种工作属性（专注时长/会议占比/分心次数/休息间隔等）计算专注质量，给用户个性化洞察 | 只有简单时间统计，可以升级 | ⭐⭐⭐ 中等 |
| **日终自动批准时间条目** | 减少用户每日审核工作量，提升使用体验 | 需要添加 | ⭐ 极小 |

### 为什么这些优先？

1. **三级分类 + 计费**：打开了新的用户群体——自由职业者需要向客户收费，他们需要准确知道每个客户花了多少时间，可以算钱。目前我们缺这个。

2. **自定义追踪规则**：解决了 AI 分类不准的痛点，让用户可以自己教系统，系统越用越准，提升粘性。

3. **分心拦截**：增强专注模式的完整性，用户开了专注就真的不会被分心网站打断。

---

## 二、中优先级 - 需要一些开发，长期价值高

这些功能让产品更完整，适合在高优先级完成后开发。

| 功能 | 价值说明 | 开发工作量 |
|------|---------|------------|
| **日历集成 (Google/Outlook)** | 自动同步会议，自动识别并记录会议时间，自动标记到对应项目 | ⭐⭐⭐⭐ 较大 |
| **时间条目批量操作** | 批量批准/分类/删除，提升每日审核效率 | ⭐⭐ 小 |
| **盈利仪表盘** | 按客户/项目/成员查看利润率，发现"过度服务"和不赚钱的项目 | ⭐⭐⭐ 中等 |
| **自定义仪表盘** | 允许用户根据自己需求配置仪表盘组件顺序和内容 | ⭐⭐⭐ 中等 |
| **工作区 Workspaces** | 支持多工作区，分离不同客户/团队的工作场景，数据隔离 | ⭐⭐⭐ 中等 |

---

## 三、低优先级 - 产品成熟后再做

这些是企业级/大规模团队需要的功能，等产品有一定用户规模再做。

| 功能 | 价值说明 |
|------|---------|
| **第三方集成 (ClickUp/Linear/Asana)** | 双向同步任务，自动同步时间条目到外部工具 |
| **自动客户报告** | 自动生成精美 PDF 报告，定期邮件发送给客户 |
| **自动发票功能** | 一键生成发票，支持自定义费率税费，PDF 邮件发送 |
| **团队容量规划** | 基于历史数据预测团队容量，防止 burnout |

---

## 四、隐私与权限 - 我们已经走在正确方向上

Rize 的隐私设计理念**我们已经完美实现**：

| Rize 隐私原则 | Merize 现状 |
|-------------|------------|
| 永远不会截图、不会记录键盘，仅读取窗口元数据 | ✅ **已实现** - Tauri 后端就是这么设计的 |
| 用户数据控制权，员工可以审核自己的数据再分享 | ✅ **架构已支持** - 团队权限设计遵循这个原则 |
| 管理者只能看到聚合的团队数据，看不到个人具体活动 | ✅ **已实现** - HR 服务只返回聚合统计 |
| 非侵入式设计，不是强制监控工具 | ✅ **符合** - 我们设计理念一致 |

## 五、我们比 Rize 多的特色卖点

| 特色 | Rize | Merize |
|------|------|--------|
| 虚拟宠物 + 效率游戏化 | ❌ 没有 | ✅ **我们独有**，这是很大的差异化 |
| 可拖拽时间块规划 | ❌ 没有 | ✅ 已有 |
| 番茄工作法原生支持 | ❌ 只有简单休息提醒 | ✅ 已有完整后端驱动 |
| 中文本地化 + 国产AI (火山引擎) | ❌ 国外产品 | ✅ 我们天生就是 |
| 价格 | $12~$40 **按月订阅** | ✅ **开源免费** |

我们的差异化很清晰：**开源免费 + 游戏化宠物 + 中文本土支持**，这是 Rize 给不了的。

---

## 六、总结与建议

### 推荐开发顺序

1. **第一步**：添加三级分类体系（客户 → 项目 → 任务）+ 可计费/非可计费标记
2. **第二步**：添加自定义追踪规则，让用户可以教 AI 识别工作
3. **第三步**：完善专注模式，添加分心拦截功能 + 计算 Focus Quality Score
4. **第四步**：添加批量操作，优化每日审核流程
5. **后续**：再考虑日历集成、盈利仪表盘等进阶功能

### 核心定位保持

Merize = **自动时间追踪 + 时间块规划 + 虚拟宠物游戏化 + 团队HR游戏化**

我们在 Rize 「自动时间追踪 + 专业计费」的基础上，**添加了独特的游戏化宠物层**，这是我们的护城河。
