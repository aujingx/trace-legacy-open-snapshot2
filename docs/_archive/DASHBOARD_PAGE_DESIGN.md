# Dashboard 仪表盘 - 页面设计文档

## 📋 概述
Dashboard 是用户打开应用看到的第一个页面，承担的核心职责是：
- **降低启动门槛**：让用户尽可能快地开始工作
- **提供即时反馈**：展示今天的进度和成就
- **智能推荐下一步**：告诉用户「现在最适合做什么」

> ✨ 最重要的设计原则：**这不是一个「监控面板」，这是一个「启动面板」**

## 💡 设计优化讨论点

### 1. 「开始专注」按钮 - 核心启动点

**当前设计**：一个普通的按钮，点击后进入 25 分钟番茄

**可以讨论的优化方向**：

#### 选项 A：多选项按钮
```
[ 🚀 开始专注 ]  ← 大按钮，默认 25 分钟
     ↓
     [ 先做 5 分钟 ]    ← 最低启动门槛
     [ 专注 25 分钟 ]
     [ 专注 50 分钟 ]
     [ 不限时 ]
```

#### 选项 B：一键继续
```
[ ⏯️ 继续上次的任务 ]   ← 最显眼的按钮
          ↓
上次你在做：「Trace 产品文档」，已专注 23 分钟
```

#### 选项 C：智能推荐
```
✨ 推荐你现在做：「准备产品演示文稿」
预估 45 分钟，是你今天最重要的事

[ 开始做这个 ]
[ 先做点别的 ]
```

**需要讨论的问题**：
- 你更倾向哪种设计哲学？「给用户选择」vs「帮用户减少选择」？
- 最低门槛的「5分钟模式」是不是应该放在最显眼的位置？
- 「继续上次」是不是应该成为默认行为？

---

### 2. 正反馈区域设计

**当前设计**：只是数字展示（「今天专注了 2 小时 13 分钟」）

**可以讨论的优化方向**：

#### 概念：「启动 Streak」
不是展示「你工作了多久」，而是展示「你成功开始了」：
```
🔥 你已经连续 3 天在上午 10 点前开始工作了！

今天：✅ 09:47 成功启动
昨天：✅ 09:15 成功启动
前天：✅ 09:32 成功启动
```

#### 概念：「今日小成就」
不是冰冷的数字，是有温度的肯定：
```
🏆 今日成就
- 上午就完成了 3 个任务 · 2小时前
- 最长一次专注持续了 72 分钟 · 真棒！
- 没有在社交媒体上浪费时间
```

**需要讨论的问题**：
- 你觉得哪种正反馈更能激励你？
- 会不会有「过度游戏化」反而让人反感的风险？
- 正反馈的频率应该多高？每完成一个任务给？还是每天总结一次？

---

### 3. 智能建议区域

**当前设计**：占位区域

**可以讨论的方向**：

#### 建议类型 1：启动助推
```
💡 看起来你有点难以开始
要不要先从最简单的事做起？
「回复邮件」大概只需要 10 分钟

[ 先做这个 ]
```

#### 建议类型 2：习惯提醒
```
📊 观察到你通常下午 3 点效率最高
要不要现在开始做最有挑战性的任务？
```

#### 建议类型 3：休息建议
```
☕ 你已经连续工作 2 小时了
站起来走 2 分钟吧，回来效率会更高
```

**需要讨论的问题**：
- 建议应该是什么样的呈现形式？卡片？侧边栏？还是状态栏的小提示？
- 建议可以被用户「拒绝」或「延后」吗？
- 建议的频率应该怎么控制才不会让人觉得被打扰？

---

### 4. Dashboard 信息密度

**当前设计**：4-5 个模块堆在一屏

**可以讨论的方向**：

#### 极简主义选项
```
[ ⏯️ 继续上次的任务 ]       ← 唯一的大按钮

你今天已经专注了 2 小时 13 分钟

[ 最近 3 个活动 ]
```

> 哲学：减少认知负荷，让启动尽可能简单

#### 丰富信息选项
```
[ 快捷操作区 ]    [ 今日成就卡 ]
[ 智能建议 ]      [ 专注热力图预览 ]
[ 待办 Top 3 ]
```

> 哲学：让用户看到全貌，掌控感更强

**需要讨论的问题**：
- 你打开 Dashboard 的时候，最想先看到什么？
- 你觉得 Dashboard 应该是极简的，还是信息丰富的？
- 如果是极简，哪些信息是必须保留的？哪些可以放到别的页面？

## 🎨 参考案例

### RescueTime Dashboard
- 优点：数据丰富，可视化好
- 缺点：太像监控报告，有压迫感

### Toggl Track Dashboard
- 优点：「继续」按钮很大，启动门槛低
- 缺点：没有智能，纯记录工具

### Rize Dashboard
- 优点：有「Focus Score」，有建议区域
- 缺点：建议偏泛泛，不够个性化
