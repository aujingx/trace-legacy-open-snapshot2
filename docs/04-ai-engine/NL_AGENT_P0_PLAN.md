# 🎙️ 自然语言任务管理 Agent - P0 实现计划

> 版本：v1.0 P0 "Quick Note"
> 目标周期：1-2 周
> 最后更新：2026-04-28

---

## 🎯 P0 目标

**做一个能用的最小可用版本，让用户可以用自然语言快速记任务。**

不做 AI 对话、不做智能重排、不做复杂规划。

就是 **一句话 → AI 提取信息 → 创建任务 → 完事**。

---

## 📋 P0 功能范围

### ✅ 做的部分

#### 1. 自然语言输入入口
- **位置**：Dashboard 页面最顶部，整个应用最显眼的位置
- **样式**：类似 ChatGPT 的输入框，左边有个 🎙️ 麦克风图标（未来语音输入用）
- **占位提示**：「告诉我你要做什么...（比如："下周要把 API 文档写完"）」
- **快捷键**：全局快捷键唤出（可选 P0 后期加）

#### 2. AI 信息提取（核心）
用户输入一句话，AI 提取 3 个字段：

| 字段 | 提取说明 | 提取不到的默认值 |
|------|---------|-----------------|
| **title** | 任务标题 | （必须要有） |
| **estimatedMinutes** | 预估时间（分钟）<br>关键词识别：「2小时」「30分钟」「1h」 | `null`（不填） |
| **dueDate** | 截止日期<br>关键词识别：「明天」「下周三」「5月1日」 | `null`（不填） |

**AI 不需要理解的（P0）：**
- ❌ 优先级
- ❌ 难度标签
- ❌ 任务依赖
- ❌ 刚性/柔性
- ❌ 自动安排到具体时间点

#### 3. 提取结果展示与确认
- AI 提取完后，输入框下方展开一个小卡片展示：
```
✅ 我理解的是：

任务：写 API 文档
预估：2 小时
截止：下周五

[ 确认添加 ]  [ 调整 ]
```
- 用户点「确认添加」→ 任务加入任务列表
- 用户点「调整」→ 展开简单的编辑器，可以手动改这 3 个字段
- **自动确认超时**：5 秒内用户没操作，自动确认添加（可设置里关掉）

#### 4. 完成反馈
- 任务添加成功后显示一个 Toast：「✅ 已添加『写 API 文档』」
- Toast 上有「撤销」按钮，5 秒内可以撤销

### ❌ P0 绝对不做的部分

1. ❌ 多轮对话、追问用户
2. ❌ 自动重排已有任务
3. ❌ 冲突检测（和已有任务时间冲突）
4. ❌ 过载保护提醒
5. ❌ 任务拆分建议
6. ❌ 刚性/柔性任务区分
7. ❌ 用户习惯学习、个性化
8. ❌ 语音输入（留接口，P1 再做）
9. ❌ 自动安排到时间线的具体时间块

### 📝 P0 的 Prompt 设计

```typescript
const PROMPT = `
你是一个任务管理助手。用户会用自然语言描述一个任务，你需要提取 3 个字段：

1. title: 任务的简短标题（中文，不超过 50 字）
2. estimatedMinutes: 预估需要多少分钟完成（数字）
3. dueDate: 截止日期，ISO 8601 格式（YYYY-MM-DD）

提取规则：
- 如果用户说了时间，比如 "2小时" "30分钟"，estimatedMinutes 填对应的分钟数
- 如果用户说了日期，比如 "明天" "下周三" "5月1日"，dueDate 填对应的日期
- 如果用户没说某个字段，那个字段填 null
- 只返回 JSON，不要其他任何解释

当前日期是：${new Date().toISOString().split('T')[0]}

用户输入：{{user_input}}

返回 JSON 格式：
{
  "title": "任务标题",
  "estimatedMinutes": 120,
  "dueDate": "2026-05-01"
}
`
```

## 🔗 和现有模块的集成

### 1. 和任务系统集成
- 调用现有的 `createTask()` API
- 只填 title、estimatedMinutes、dueDate 三个字段
- 其他字段用默认值：priority = 3, status = todo

### 2. 和 Dashboard 集成
- NLTaskInput 组件放在 Dashboard 最顶部
- 任务添加成功后，自动刷新 Dashboard 的任务列表

### 3. 和事件总线集成
- 任务创建成功后发布 `TaskCreated` 事件
- 未来其他模块可以监听这个事件做后续处理（P1 再用）

## 🗺️ 完整路线图：从 P0 到 P3

### 🎯 P0：Quick Note 快速备忘录（2-3 天）
**目标**：能用，一句话快速记任务
- ✅ 自然语言输入框放在 Dashboard 顶部
- ✅ 提取 3 个核心字段：标题、预估时间、截止日期
- ✅ 结果展示 + 手动调整
- ✅ Toast 反馈 + 5 秒撤销
- ✅ 5 秒无操作自动确认

---

### 🎯 P1：Smart Assistant 智能助理（1-2 周）
**目标**：开始有"智能"的感觉，能感知上下文

#### P1.1 感知已有安排
- 安排新任务时检查是否和已有任务时间冲突
- 简单的冲突提醒：「你周三下午已经有 2 个任务了，还要加进去吗？」

#### P1.2 更多信息提取
- 提取优先级关键词：「紧急」「重要」「不急」
- 提取难度关键词：「很简单」「有点难」「需要花点时间」
- 自动设置情绪标签 😊😐😰

#### P1.3 简单的重排命令
- 支持自然语言的调整命令：
  - 「把 A 移到明天」
  - 「A 和 B 换一下」
  - 「把 C 推迟一周」

#### P1.4 过载保护提醒
- 当天安排超过用户历史平均值的 120% 时，温和提醒：
  - 「你今天已经安排了 6 小时专注，再加这个就 8 小时了，比平时多了 50% 😅」

---

### 🎯 P2：Dynamic Planner 动态规划师（2-3 周）
**目标**：真正的 Agent，可以帮用户管理变化

#### P2.1 刚性 / 柔性任务区分
- 关键词识别「必须」「一定要」「不能改」→ 刚性任务
- 重排时刚性任务不动，只动柔性任务
- 给用户展示「调整说明」：哪些动了，哪些没动，为什么

#### P2.2 自动重排引擎
- 用户说「来不及了」「这个做不完了」→ 自动把低优先级柔性任务往后移
- 自动给出 2-3 个重排方案供选择
- 所有重排操作可一键撤销

#### P2.3 任务依赖关系
- 识别「做完 A 才能做 B」的表述
- 自动建立依赖关系，安排时间时自动保证顺序
- 前置任务推迟时，后置任务自动跟着推

#### P2.4 任务拆分建议
- 识别太大的任务（预估 > 4 小时）
- 自动建议拆分：「这个任务看起来要 8 小时，要不要拆成 4 个 2 小时的小任务？」
- 给出拆分建议，用户确认后自动生成子任务

---

### 🎯 P3：Personal Coach 私人教练（1-2 个月）
**目标**：真正理解用户，个性化优化

#### P3.1 用户习惯学习
- 记录用户的历史数据：
  - 每天平均能专注多久？
  - 什么时间段效率最高？
  - 通常低估还是高估自己的速度？
  - 喜欢把难的任务放什么时候做？
- 安排任务时自动应用这些习惯
- 预估时间自动应用用户的偏差系数（比如总是低估 30% → 自动 × 1.3）

#### P3.2 深度对话式规划
- 大任务启动多轮对话模式：「我们来聊聊毕业论文这个计划」
- AI 主动提问关键信息
- 讨论完后自动生成完整的执行计划 + 任务列表 + 时间线
- （就是 PRODUCT_IDEAS.md 里想法 001 的完整实现）

#### P3.3 动态调整与反馈循环
- 任务实际完成时间和预估不一样时，自动调整未来的预估
- 观察用户实际的工作模式，持续优化推荐算法
- 每周给用户一个温和的总结：「这周你安排的任务平均完成了 70%，下周建议少安排 20% 会更从容」

#### P3.4 任务性质识别与智能搭配
- 识别「深度专注任务」和「缓冲任务」（打电话、回邮件等）
- 自动穿插安排，张弛有度
- 缓冲任务自动安排到用户容易犯困的时间段

## 🎨 设计要点

1. **极简、极速**：整个流程 2 秒内完成，不要打断用户
2. **犯错了也没关系**：AI 提取错了，用户可以撤销/修改，不要怕错
3. **渐进式披露**：P0 看起来就是个「快速输入框」，P1 再慢慢加智能功能，用户不会有心理负担
4. **所有操作可撤销**：至少保留 5 秒的撤销窗口

---

## 🔍 风险点

1. **AI 提取准确率**：中文自然语言的时间表达千奇百怪，P0 不追求 100% 准确，80% 就够了，错了用户可以改
2. **API 成本**：每次调用几分钱，对个人用户来说可以忽略不计
3. **隐私问题**：用户的任务内容会发给 OpenAI，需要在设置里加开关，默认关闭让用户自己选开不开
