# 任务管理 - 页面设计文档 v2.0

> **最后更新**: 2026-05-02
> **设计哲学**: 我们不做「更好的 GTD」，我们做「让你愿意创建任务，并且愿意开始做任务」的任务管理。
>
> **核心矛盾**: 创建任务的门槛越高，用户就越不想创建任务；任务列表越长，用户越容易产生「我什么都不想做」的麻木感。

---

## 📊 设计评估与优化方向

### 当前状态评估 (v1.0)

| 维度 | 得分 | 说明 |
|------|------|------|
| 核心创新点 (firstStep/情绪标签/AI 推荐) | 9/10 | 行业领先，直击拖延心理痛点 |
| 设计哲学 | 10/10 | 「反 GTD + 共情」的定位非常精准 |
| 行业 UX 基线对齐 | 5/10 | 缺少主流产品驯化的基础交互，导致「别扭感」 |
| 功能落地性 | 7/10 | 很多好设计缺少明确的 UI 入口和交互流程 |
| 可视化程度 | 4/10 | 进度、雪球效应缺少具象化展示 |

### v2.0 优化重点

1. **P0 必须先做**: 对齐行业 UX 基线，消除「别扭感」
2. **P1 核心差异化**: AI 能力可视化，让用户感知到智能
3. **P2 情感连接**: 完成正反馈 + 拖延共情闭环

### 1. 「一行输入」快速创建任务

#### 当前问题
创建任务需要填 6 个字段（标题、优先级、预估时间、截止日期、第一步、情绪标签），用户望而却步：
> 心理活动：「好麻烦，等下再填吧」→ 然后就忘了。

#### 最终设计方案
**顶部永远有一个快速输入框（行业标准）**：

```
┌─────────────────────────────────────────────────────────┐
│ 🔍 今天要做什么？按回车快速创建...                        │
│                                                         │
│ 💡 试试自然语言："明天下午3点 写报告 P1 45分钟"          │
└─────────────────────────────────────────────────────────┘
```

#### 交互规则
- 用户只输入标题，按回车 → **立即创建成功**
- 其他所有字段（优先级、预估时间等）都是**可选的，事后编辑**
- 支持自然语言智能解析（AI 自动识别）

#### 自然语言解析规则
```
输入："明天下午3点 写报告 P1 45分钟 第一步: 打开文档"
  ↓ 自动解析 ↓
{
  title: "写报告",
  dueDate: "2026-05-03 15:00",
  priority: 1,
  estimatedMinutes: 45,
  firstStep: "打开文档"
}
```

#### 技术实现要点
- 优先级识别：`P[0-5]` 或 `p[0-5]`
- 时间识别：`30分钟` / `45m` / `1h` / `1.5小时`
- 日期识别：`今天` / `明天` / `下周一` / `5-20` / `2026-05-20`
- 第一步识别：`第一步: xxx` 或 `第一步 xxx`

### 2. 任务卡片 UX 重构（双复选框问题）

#### 当前问题
任务卡片左侧同时有两个复选框：
1. 多选复选框（批量操作）
2. 完成复选框（标记完成）

> 用户困惑：「我想打勾完成，应该点哪个？」

#### 最终设计方案（行业标准）

**单复选框原则**：
```
┌─────────────────────────────────────────────────────┐
│ ✓  任务标题                                      │
│    🐣 第一步：打开文档                              │
│    P1  😊  📅 今天  ⏱️ 45分钟                      │
│                                      [▶️] [😔]    │ ← 悬停显示操作
└─────────────────────────────────────────────────────┘
```

#### 交互规则
| 操作 | 结果 |
|------|------|
| 点击左侧复选框 | 标记完成 / 取消完成 |
| 点击卡片其他区域 | 打开右侧详情面板（编辑） |
| 长按卡片 0.5 秒 | 进入多选模式 |
| 右滑卡片 | 显示「删除」「推迟到明天」 |
| 悬停卡片 | 显示「开始专注」「我不想做」按钮 |

#### 批量操作入口
- 顶部导航栏有「批量操作」按钮，点击进入多选模式
- 长按任意卡片也可进入多选模式

### 3. 默认视图：今日优先

#### 当前问题
打开 Planner 显示所有任务，包括下周、下个月甚至没有截止日期的任务：
> 心理活动：「怎么有这么多事要做... 先刷会儿手机吧。」→ 信息过载导致逃避。

#### 最终设计方案

**默认只显示「今日 + 逾期」任务，其他折叠**（所有主流任务管理产品的标准做法）：

```
┌─────────────────────────────────────────────────┐
│ 📅 今天 (2026年5月2日 星期五)                   │
│ 已完成 3/8 个任务  [████████░░░░] 37.5%         │
├─────────────────────────────────────────────────┤
│ [✓] 给设计稿加标注                              │
│ [✓] 回复客户邮件                                │
│ [✓] 开周会                                      │
│ [ ] 完成项目报告 P1 ⏱️ 45分钟                   │
│ [ ] 准备演示文稿 P2 ⏱️ 30分钟                   │
│                                                 │
│ 👇 还有 12 个任务不在今天                        │
│ [ 点击展开其他任务 ]                             │
└─────────────────────────────────────────────────┘
```

#### 视图切换
顶部仍保留 4 种视图切换：`列表` / `看板` / `表格` / `日历`

### 4. AI 智能拆解大目标

#### 设计背景
用户有一个大目标，但不知道从何开始：
> 「我要在两周内完成项目报告，但一想到要做这么多事，我就不想开始。」

#### 最终设计方案

**AI 拆解入口**：
```
┌─────────────────────────────────────────────────┐
│ 🎯 有一个大目标不知道怎么开始？                   │
│                                                 │
│ [ 输入你的目标，比如：两周内完成项目报告      ]  │
│ [ ✨ AI 智能拆解 ]                               │
└─────────────────────────────────────────────────┘
```

**AI 拆解结果展示（可编辑）**：
```
🎯 目标：两周内完成项目报告
进度：[░░░░░░░░░░] 0%

AI 帮你拆解成 5 个可执行的小任务：

1. [ ] 收集历史数据和参考资料
   ⏱️ 预估 30 分钟  ·  P2  ·  今天就可以开始

2. [ ] 撰写大纲和章节结构
   ⏱️ 预估 45 分钟  ·  P2  ·  完成第1步后开始

3. [ ] 填充各章节内容
   ⏱️ 预估 120 分钟  ·  P1  ·  分 3 次完成

4. [ ] 排版和格式调整
   ⏱️ 预估 45 分钟  ·  P2  ·  完成第3步后开始

5. [ ] 最终检查和提交
   ⏱️ 预估 20 分钟  ·  P1  ·  截止日前一天

[ 💾 全部保存到任务列表 ]  [ ✏️ 手动调整 ]
```

#### AI Prompt 模板
```typescript
const PROMPT = `
把这个目标拆解成 3-7 个具体、可执行的小任务：
目标：${goal}

拆解原则：
1. 第一个任务必须能在 15 分钟内完成（降低启动门槛）
2. 每个任务标题不超过 20 个字
3. 按执行顺序排列，有明确的依赖关系
4. 合理预估每个任务的时间（10-120 分钟）
5. 标注建议的优先级：P1（重要）/ P2（普通）/ P3（可延后）

请用 JSON 格式返回：
{
  "tasks": [{
    "title": "任务标题",
    "estimatedMinutes": 30,
    "priority": 2,
    "firstStep": "第一步具体做什么",
    "dependency": "前置任务序号，没有则为 0"
  }]
}
`
```

### 5. AI 推荐理由可视化

#### 设计背景
7 维度推荐算法已经实现，但用户看不到「为什么这个任务排在前面」，感受不到智能。

#### 最终设计方案

**每个任务显示推荐理由标签**：
```
┌─────────────────────────────────────────────────────┐
│ ✓  准备产品演示                                  │
│    🏷️ 优先级高  ·  ⚠️ 今天截止  ·  已开始        │ ← 推荐理由
│    🐣 第一步：打开 Keynote                         │
│    P1  😰  📅 今天  ⏱️ 45分钟                      │
└─────────────────────────────────────────────────────┘
```

#### 推荐理由标签库

| 标签 | 触发条件 |
|------|---------|
| 🏷️ 优先级高 | P0 或 P1 |
| ⚠️ 已逾期 | 截止日期 < 今天 |
| 📅 今天截止 | 截止日期 = 今天 |
| ⏰ 即将截止 | 3 天内截止 |
| 🚀 接近完成 | 已完成 >= 80% |
| ▶️ 已开始 | 有已记录的时间 |
| 🧠 符合你的习惯 | 这个时间段经常做这类任务 |
| 🐣 轻松易启动 | 情绪标签 = easy，且时间 <= 15 分钟 |

### 6. 目标树可视化

#### 设计背景
用户看不到大目标和小任务之间的关系，缺少掌控感和成就感。

#### 最终设计方案

**新增「目标」视图标签**：
```
列表  看板  表格  日历  [🎯 目标]
```

**目标树展示**：
```
🎯 Q2 产品上线
进度：[██████░░░░] 60%
├─ ✅ 完成需求评审 (100%)
├─ ✅ UI 设计稿 (100%)
├─ 🟨 前端开发 (70%)
│  ├─ ✅ 登录注册页
│  ├─ ✅ 任务管理页
│  └─ [ ] 统计仪表盘页面  ⏱️ 剩余 2 小时
└─ [ ] 后端 API 开发 (20%)
   ├─ [ ] 用户认证接口
   └─ [ ] 任务数据接口
```

#### 交互规则
- 点击目标可以展开/折叠子任务
- 拖拽任务可以归属到任意目标下
- 子任务完成自动更新父目标进度
- 支持 AI 一键拆解目标（见上一节）

### 7. 任务完成正反馈闭环

#### 设计背景
点击「完成」→ 任务划掉 → 消失，没有成就感。

#### 最终设计方案

**完成庆祝弹窗（右下角弹出，3 秒后自动消失）**：
```
          🎉 太棒了！
「准备产品演示」完成了！
预估 45 分钟，实际花了 38 分钟，快了 7 分钟！

这是你今天完成的第 4 个任务。
你已经专注了 2 小时 13 分钟。

          继续加油！ 🌟
```

#### 雪球效应可视化（页面顶部）
```
🏔️ 今日雪球进度
[████████████░░░░░░░░░░] 4/10

💪 连续专注 4 天了，继续保持！
```

### 8. 「我不想做」拖延共情完整闭环

#### 设计背景
有时候用户明知道应该做某个任务，但就是不想开始。不是任务难，是「开始」这个动作难。

#### 最终设计方案

**悬停显示「😔 我不想做」按钮，点击弹出**：
```
            我理解 😌
不想开始是很正常的，不用责怪自己。

👇 你可以试试：

┌─────────────────────────────────┐
│ ⏱️ 就做 5 分钟                  │
│    开启一个 5 分钟的专注，做完   │
│    就可以停下来，没有压力。      │
├─────────────────────────────────┤
│ 🐣 先做个更简单的                │
│    推荐一个 10 分钟就能做完的    │
│    简单任务，先热个身。          │
├─────────────────────────────────┤
│ 📅 明天再说                      │
│    自动把截止日期改到明天，       │
│    今天就放过自己。              │
└─────────────────────────────────┘
```
