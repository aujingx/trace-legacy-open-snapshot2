# 🤖 AI 智能事件推荐系统

> **设计原则**：从「简单的优先级排序」升级为「综合多维因素的智能推荐」，让 AI 学习用户习惯，推荐最适合当前场景的事件。
>
> **核心概念**：只有「事件」这一种实体。「任务」「时间块」只是它的两种显示状态，推荐系统对它们一视同仁。

---

## 🎯 核心设计思想

### 问题背景
- ❌ 简单按 P1 → P2 → P3 排序太死板
- ❌ 不考虑任务实际需要的时间
- ❌ 不考虑用户当前的状态和习惯
- ❌ 不考虑截止日期的紧迫性

### 解决方案
AI 推荐系统综合考虑 **7 大维度**，为每个任务计算综合得分，按得分排序：

| 维度 | 权重（默认） | 说明 |
|------|-------------|------|
| **任务优先级** | 35% | P0 > P1 > P2 > P3 > P4 |
| **时间紧迫性** | 25% | 已逾期 > 今天截止 > 明天截止 > 一周内 |
| **时长适配度** | 15% | 任务时长与当前可用时间的匹配程度 |
| **用户习惯** | 10% | 用户习惯在这个时间段做的任务类型 |
| **任务进度** | 8% | 接近完成的任务优先推荐 |
| **上下文匹配** | 5% | 与用户当前活动相关的任务优先 |
| **情感标签** | 2% | easy > neutral > resist，阻力小的任务优先 |

## ⚙️ 推荐配置系统

### 三种预设模式

| 模式 | 适用场景 | 特点 |
|------|---------|------|
| **🤖 AI 自动** | 大多数用户 | 智能平衡所有因素，推荐最适合的任务 |
| **📊 均衡推荐** | 需要平衡多维度时 | 优先级和紧迫性各占 30% |
| **⭐ 优先级别优先** | 严格按级别推进 | 优先级占 60%，几乎只看 P1/P2 |

### 用户自定义权重

用户可以在设置中微调每个维度的权重：

```typescript
interface RecommendationWeights {
  priority: number        // 优先级权重：0-1
  urgency: number         // 紧迫性权重：0-1
  durationFit: number     // 时长适配权重：0-1
  habit: number           // 用户习惯权重：0-1
  progress: number        // 任务进度权重：0-1
  contextMatch: number    // 上下文匹配权重：0-1
  emotionalTag: number    // 情感标签权重：0-1
}
```

**设计约束**：
- 所有权重总和建议为 1.0
- 每个权重最小为 0，最大为 1.0
- 至少有一个权重大于 0
- 提供「一键恢复默认」按钮

## 🧠 机器学习用户习惯

### 习惯数据结构

```typescript
interface UserHabitRecord {
  // 时间段（小时 0-23）-> 任务类别 -> 完成次数
  timeCategoryCounts: Record<number, Record<string, number>>

  // 星期几 -> 任务类别 -> 完成次数
  weekdayCategoryCounts: Record<number, Record<string, number>>

  // 任务类别 -> 平均完成时长
  categoryAverageDuration: Record<string, number>

  // 最近完成的任务 ID 列表（用于去重推荐）
  recentCompletedTaskIds: string[]
}
```

### 学习机制

**每次用户完成任务时自动记录**：
1. 完成的时间（小时）
2. 星期几
3. 任务类别
4. 实际完成时长

**学习效果示例**：
- 用户经常在早上 9 点写代码 → 9 点推荐「开发」类任务
- 用户经常在周一开会 → 周一推荐「会议」相关任务
- 用户做「设计」类任务平均需要 45 分钟 → 推荐时考虑时长匹配

**数据持久化**：
- 存储在 localStorage，支持跨会话
- 提供「清除习惯数据」的重置选项
- 最多记录最近 50 个完成的任务

## 📊 评分算法详解

### 1. 优先级评分 (Priority Score)

```typescript
const priorityScores = {
  0: 100,  // P0 - 最高优先级，紧急且重要
  1: 80,   // P1
  2: 60,   // P2
  3: 40,   // P3
  4: 20,   // P4
}
```

### 2. 紧迫性评分 (Urgency Score)

| 截止时间 | 得分 |
|---------|------|
| 已逾期 | 100 |
| 今天截止 | 95 |
| 明天截止 | 85 |
| 3天内 | 70 |
| 一周内 | 50 |
| 两周内 | 30 |
| 两周以后 | 15 |
| 没有截止日期 | 30 |

### 3. 时长适配评分 (Duration Fit Score)

假设用户当前可用时间为 60 分钟：
| 任务预估时长 | 得分 |
|------------|------|
| 30-60 分钟 | 100 |
| 15-30 分钟 | 60 |
| 60-90 分钟 | 75 |
| > 90 分钟 | 30 |

### 4. 用户习惯评分 (Habit Score)

- 基础分：50 分
- 当前小时完成过同类任务：+ 次数 × 5 分（最多 30 分）
- 今天是星期几，当天完成过同类任务：+ 次数 × 3 分（最多 20 分）
- 最高分：100 分

### 5. 任务进度评分 (Progress Score)

| 状态 | 得分 |
|------|------|
| 完成度 >= 80% | 100 |
| 完成度 >= 50% | 80 |
| 已开始但 < 50% | 60 |
| 未开始 | 40 |

### 6. 情感标签评分 (Emotional Score)

| 标签 | 得分 |
|------|------|
| easy (易启动) | 100 |
| neutral (中性) | 70 |
| resist (有阻力) | 30 |

## 🎁 推荐理由解释

每个推荐任务都会显示「为什么推荐给你」的解释标签：

| 标签 | 触发条件 |
|------|---------|
| 优先级高 | 优先级得分 >= 80（P0/P1） |
| 已逾期 | 截止日期已过 |
| 今天截止 | 今天是截止日期 |
| 即将截止 | 3 天内截止 |
| 接近完成 | 已完成 80% 以上 |
| 已开始 | 任务有进展但未完成 |
| 符合你的习惯 | 习惯得分 >= 70 |
| 轻松易开始 | 情感标签为 easy |

**示例展示**：
```
📌 重构登录模块
   理由：优先级高 · 今天截止 · 已开始
```

## 🏗️ 技术实现架构

> 🎯 **统一数据模型原则**：推荐系统只有「事件」这一种概念。
> - 一个事件被安排了时间 → 在 Timeline 显示，叫「时间块」
> - 同一个事件没安排时间 → 在 Tasks 显示，叫「任务」
> - 它们共享同一个 ID、同一份数据、同一个推荐算法
> - 在 Dashboard 看到的推荐 = 在 Tasks 页面看到的排序 = 在 Timeline 看到的安排

### 文件结构

```
src/
├── services/
│   └── taskRecommendation.ts      # 核心推荐算法
│       ├── scoreAndSortTasks()    # 计算所有任务得分并排序
│       ├── getTopRecommendations() # 获取前 N 个推荐
│       ├── recordTaskCompletion()  # 记录任务完成，学习习惯
│       ├── getUserHabits()        # 获取习惯统计
│       ├── resetUserHabits()      # 重置习惯数据
│       └── getPresetWeights()     # 获取预设权重配置
└── store/
    └── useAppStore.ts              # 全局状态集成
        ├── recommendationMode      # 当前推荐模式
        ├── recommendationWeights   # 权重配置
        ├── getRecommendedTask()    # 获取单个推荐任务
        └── getRecommendedTasks()   # 获取多个推荐任务
```

### 公开 API 使用示例

```typescript
// 1. 获取推荐任务（在 Dashboard 中使用）
const topTasks = useAppStore.getState().getRecommendedTasks(3)

// 2. 修改推荐模式
useAppStore.getState().setRecommendationMode('priority_only')

// 3. 自定义权重
useAppStore.getState().setRecommendationWeights({
  priority: 0.5,
  urgency: 0.3,
  habit: 0.2,
})

// 4. 重置为默认权重
useAppStore.getState().resetRecommendationWeights()

// 5. 完成任务后记录习惯（AI 学习）
import { recordTaskCompletion } from '../services/taskRecommendation'
await updateTask(task.id, { status: 'completed' })
recordTaskCompletion(task)  // AI 学习用户的完成习惯
```

### 数据持久化

```
LocalStorage Keys:
- trace-recommendation-mode      # 推荐模式
- trace-recommendation-weights   # 权重配置
- trace-user-habits             # 用户习惯数据
```
