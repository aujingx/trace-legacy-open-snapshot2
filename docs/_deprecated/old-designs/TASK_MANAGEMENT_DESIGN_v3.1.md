## 模式切换体系设计（v3.1 新增）
## ============================================================

### 核心原则：两个独立维度，互不干扰

| 维度 | 切换位置 | 选项 |
|------|---------|------|
| **视图类型** | 顶部右侧（沿用现有位置） | 列表 / 看板 / 表格 / 日历 / 目标 |
| **信息密度** | 任务列表顶部（新增加） | 单任务 / 今日 / 全部 |

### 完整界面布局

```
┌─────────────────────────────────────────────────────────────────┐
│ 🔍 Trace                                                         │
│                                                         ┌─────┐ │
│  视图： [📋 列表] [📊 看板] [📝 表格] [📅 日历] [🎯 目标]│ ⚙️  │ │
│  └─────────────────────── 第一级：视图类型 ───────────────────┘ │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  显示： [🎯 单任务]  [📋 今日]  [📂 全部]                       │
│  └──────────────────── 第二级：信息密度 ──────────────────────┘ │
│                                                                 │
│  ... 任务列表 ...                                               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 渐进式发现机制

| 用户阶段 | 显示的密度选项 | 说明 |
|---------|---------------|------|
| 第一次打开 | [📋 今日] [📂 全部] | 只显示最常用的 2 个，避免 overwhelm |
| 使用 7 天后 | [🎯 单任务] [📋 今日] [📂 全部] | 解锁单任务模式，附带气泡提示 |
| 高级用户 | 全部选项 + 键盘快捷键 | `Shift + D` 快速循环切换 |

### 各视图下的密度选项适配

| 视图类型 | 单任务 | 今日 | 全部 | 说明 |
|---------|--------|------|------|------|
| 📋 列表 | ✅ | ✅ | ✅ | 全部可用 |
| 📊 看板 | ❌ | ✅ | ✅ | 看板不适合单任务模式 |
| 📝 表格 | ❌ | ❌ | ✅ | 表格就是全局管理用的 |
| 📅 日历 | ❌ | ❌ | ✅ | 日历本身就是按日期展示的 |
| 🎯 目标 | ❌ | ❌ | ✅ | 目标树就是全局视角 |

> 智能适配：用户切换到看板/表格/日历/目标视图时，「信息密度」切换器自动隐藏或置灰

## 单任务模式 vs 专注模式 边界清晰化
## ============================================================

### 核心区别对比

| 维度 | 单任务模式 | 专注模式页面 |
|------|-----------|-------------|
| **用户动机** | 「不知道接下来做什么，帮我选一个」 | 「我已经决定要做这件事了，现在开始专注」 |
| **核心问题** | 解决选择焦虑和信息过载 | 解决专注过程中的干扰 |
| **状态** | 浏览 + 决策状态 | 执行 + 沉浸状态 |
| **时间感** | 没有倒计时，没有压力 | 有明确的倒计时，有时间压力 |
| **典型停留时长** | 10 秒 - 2 分钟 | 25 分钟 - 2 小时 |
| **打断成本** | 打断了也无所谓，随时可以切走 | 打断成本很高，需要保护心流 |
| **所在页面** | Planner 页面内的一个密度选项 | 独立的 Focus 页面 |

### 联动关系设计

```
┌─────────────┐      点击「开始专注」      ┌─────────────┐
│             │ ─────────────────────────> │             │
│  单任务模式  │                            │  专注模式   │
│  (Planner)  │                            │  (Focus)   │
│             │ <───────────────────────── │             │
└─────────────┘      专注完成/放弃         └─────────────┘
       │                                        │
       └──────────────┬───────────────────────┘
                      │
               共享同一个状态
               当前任务 + 已记录时间
```

### 完整交互流程

```
1. 用户打开 Planner
   → 默认显示今日视图

2. 用户觉得任务太多，看着烦
   → 点击「单任务」密度切换
   → 进入单任务模式，只显示一个 AI 推荐的任务

3. 用户看了看这个任务，觉得还行
   → 点击「开始专注」按钮
   → ✨ 平滑过渡动画
   → 进入全屏专注模式页面
   → 开始倒计时 + 计时

4. 专注完成
   → 点击「完成」
   → 显示庆祝弹窗
   → ✨ 平滑回到 Planner 的单任务模式
   → 自动推荐下一个任务

5. 如果用户在单任务模式里直接打勾完成
   → 不会进入专注模式
   → 直接显示完成反馈
   → 推荐下一个任务
```

## P0 阶段功能详细设计（第一阶段：消除别扭感）
## ============================================================

### 1. 任务卡片双复选框重构

**当前问题**：左侧同时有「多选复选框」和「完成复选框」，用户困惑

**修改后**：
- 只保留一个「完成」复选框，点击直接标记完成
- 多选模式入口：长按卡片 0.5 秒，或顶部「批量操作」按钮
- 卡片悬停显示：「开始专注」「我不想做」按钮

---

### 2. 顶部一行快速创建任务

**界面**：
```
┌─────────────────────────────────────────────────────────┐
│ 🔍 今天要做什么？按回车快速创建...                      │
│                                                         │
│ 💡 试试自然语言："明天下午3点 写报告 P1 45分钟"          │
└─────────────────────────────────────────────────────────┘
```

**自然语言解析规则**：
```typescript
// 优先级：P0-P5
const priorityRegex = /P([0-5])/i

// 时间：30分钟 / 45m / 1h / 1.5小时
const durationRegex = /(\d+(?:\.\d+)?)\s*(分钟|m|小时|h)/i

// 日期：今天 / 明天 / 下周一 / 5-20 / 2026-05-20
const dateRegex = /(今天|明天|下周[一二三四五六日]|\d{1,2}-\d{1,2}(?:-\d{4})?)/i

// 第一步：第一步 xxx / 第一步：xxx / 先 xxx
const firstStepRegex = /(?:第一步[:：]?\s*|先\s*)(.+)/i
```

**交互规则**：
- 输入标题 + 回车 → **立即创建成功**
- 其他字段（优先级/时间/日期/第一步）自动解析填充
- 所有字段事后都可以再编辑

---

### 3. 默认今日视图 + 其他任务折叠

**顶部进度条**：
```
┌─────────────────────────────────────────────────────────┐
│ 📅 今天 (2026年5月2日 星期五)                            │
│ 已完成 3/8 个任务  [████████░░░░] 37.5%                  │
│                                                         │
│ [ 🐣 找个简单的先做 ]  [ ⏱️ 就试5分钟 ]  [ 🤷 明天再说 ] │
└─────────────────────────────────────────────────────────┘
```

**今日任务列表**：
- 只显示截止日期为今天 + 逾期的任务
- 其他任务默认折叠，显示「还有 12 个任务不在今天，点击展开」
