## 一、全局一致性规范（v4.5 新增）
## ============================================================

### 1.1 核心原则：一个组件，一个设计，到处复用

**所有同类元素，在任何页面，外观、交互、数据结构必须 100% 一致**

> ❌ 错误的做法：
> - Planner 的任务卡片长这样
> - Timeline 的待安排任务卡片长那样
> - 编辑弹窗又是另一种设计
>
> ✅ 正确的做法：
> - 同一个 TaskCard 组件，所有页面复用
> - 同一个任务面板，添加/详情/编辑都用它
> - 同一种时间块，Timeline 和 Analytics 用一样的配色

---

### 1.2 Timeline 页面一致性规范

#### ✅ 1. 待安排任务列表（左侧）

```
┌─────────────────────────────────┐
│ 📋 待安排                       │
├─────────────────────────────────┤
│                                 │
│  ┌───────────────────────────┐ │
│  │ 准备产品演示          ⋯  │ │  ← 这个卡片 = TaskCard 组件
│  │ 📅 无截止日期  ⏱️ 45分钟   │ │     和 Planner 页面完全一致
│  └───────────────────────────┘ │
│                                 │
│  ┌───────────────────────────┐ │
│  │ 写周报                ⋯  │ │  ← 也是同一个 TaskCard
│  └───────────────────────────┘ │
│                                 │
└─────────────────────────────────┘
```

**必须保持一致：**
- ✅ 卡片圆角 = `RADII.lg`（和全局一致）
- ✅ 卡片阴影 = `SHADOWS.cardSmall`（和全局一致）
- ✅ 间距 = 16px（和 Planner 列表间距一致）
- ✅ hover 效果 = 右移 2px + 阴影增强（和全局一致）
- ✅ 点击「⋯」菜单 = 打开**统一任务面板**（和 Planner 点击卡片一致）
- ✅ 优先级圆点颜色 = 和全局颜色变量一致

---

#### ✅ 2. Timeline 时间块（右侧）

```
┌─────────────────────────────────────────────────────────┐
│  09:00  ┌──────────┐                                      │
│  10:00  │ 深度工作 │  ← 时间块颜色                      │
│  11:00  │ (45分钟) │     必须和 Analytics 饼图          │
│  12:00  └──────────┘     以及全局分类配色 100% 一致     │
│          ┌──────┐                                        │
│  13:00  │ 午休  │                                        │
│  14:00  └──────┘                                        │
└─────────────────────────────────────────────────────────┘
```

**必须保持一致：**
- ✅ 时间块的颜色 = 和 Analytics 饼图的分类颜色完全相同
- ✅ 时间块的圆角 = `RADII.md`（和全局按钮圆角一致）
- ✅ hover 效果 = 轻微放大 + 边框高亮（和全局卡片 hover 一致）
- ✅ 点击时间块 = 打开**统一任务面板**的「查看详情」状态

---

#### ✅ 3. 拖拽交互一致性

```
拖拽任务到时间轴上：

- 拖拽过程中的半透明效果 = 和全局拖拽效果一致
- 放置成功的动画 = 和全局成功反馈一致
- 放置失败的回弹动画 = 和全局错误反馈一致
```

---

### 1.3 任务编辑/详情弹窗一致性

**所有地方打开的任务弹窗，都必须是同一个组件：**

| 入口位置 | 打开的弹窗 | 状态 |
|---------|-----------|------|
| Planner 点击任务卡片 | 统一任务面板 | 查看详情 |
| Planner 点击「编辑」 | 统一任务面板 | 编辑模式 |
| Timeline 点击待安排任务 | 统一任务面板 | 查看详情 |
| Timeline 点击时间块 | 统一任务面板 | 查看详情 |
| Timeline 右键 → 编辑 | 统一任务面板 | 编辑模式 |
| FAB 点击「手动添加」 | 统一任务面板 | 添加模式 |
| Dashboard 点击任务 | 统一任务面板 | 查看详情 |

**7 个入口，同一个组件，不同状态，100% 一致的体验**

---

### 1.4 必须保持一致的设计 Token

**所有组件必须引用全局变量，绝对不能硬编码颜色/尺寸：**

| 设计元素 | 全局变量 | 说明 |
|---------|---------|------|
| 主色 | `var(--color-blue)` | 所有按钮、选中状态、强调 |
| 成功色 | `var(--color-green)` | 完成、成功、正向反馈 |
| 警告色 | `var(--color-orange)` | 提醒、逾期 |
| 文字主色 | `var(--color-text-primary)` | 所有标题、重要文字 |
| 文字次色 | `var(--color-text-muted)` | 所有次要信息、时间 |
| 卡片圆角 | `RADII.lg` | 所有卡片 |
| 按钮圆角 | `RADII.md` | 所有按钮 |
| 卡片阴影 | `SHADOWS.cardSmall` / `SHADOWS.card` | 默认 / hover |
| 动画时长 | `ANIMATIONS.normal` | 所有过渡效果 |
| 间距 | 8px / 12px / 16px / 24px | 4px 网格系统 |

---

### 1.5 交互一致性 Checklist

开发完成后，用这个 Checklist 检查：

- [ ] 所有页面的 TaskCard 看起来一模一样
- [ ] 所有页面的 TaskCard hover 效果一样
- [ ] 所有页面点击任务，打开的是同一个弹窗
- [ ] 弹窗的标题、按钮、字段排列完全一致
- [ ] Timeline 的时间块颜色，和 Analytics 饼图的对应分类颜色完全相同
- [ ] 所有按钮的圆角、间距、阴影一致
- [ ] 所有输入框的外观一致
- [ ] 所有下拉选择器的外观一致
- [ ] 所有成功/错误提示的样式一致
- [ ] FAB 在三个页面（Planner/Timeline/Dashboard）的位置、大小、外观完全一致

---

### 1.6 为什么一致性这么重要

1. **用户学习成本最低**：学会一次，到处能用
2. **开发效率最高**：一次开发，到处复用
3. **Bug 最少**：改一个地方，所有地方都修好
4. **品牌感最强**：用户会感觉到「这是一个设计精良的产品」

## 二、数据分析页面设计（v4.3 新增）

### 1.1 设计哲学：正向鼓励，只看好的一面

**核心原则：数据是用来鼓励的，不是用来审判的**

> ❌ 错误的方式：「你今天只专注了 2 小时，比昨天少了 50%」
> ✅ 正确的方式：「太棒了！你今天已经专注了 2 小时，比上周平均水平提升了 15% 🎉」

**设计要点：**
1. 永远强调「进步」和「成就」，弱化「不足」
2. 去掉所有可能引发焦虑的对比（比如「比别人少」）
3. 只和自己的历史对比，而且只看好的变化
4. 下载功能移除，聚焦于数据带来的正向感受

## 二、六大设计原则
## ============================================================

### 张一鸣原则（字节跳动视角）

**原则 1：AI 是用来做减法的，不是用来做展示的**
> AI 的价值是帮用户省脑力，不是展示自己有多聪明。
> - 坏的 AI：给你 10 个选项让你选
> - 好的 AI：直接告诉你「现在就做这个」

**原则 2：每一次交互都是数据**
> 用户点击「我不想做这个」不是失败，而是获得了一个极高价值的用户偏好信号。
> - 收集：什么类型的任务、在什么时间段、用户不想做
> - 学习：用户的能量曲线是什么样的
> - 优化：下次推荐更匹配的任务

**原则 3：Less = More，Slow = Fast**
> 信息越少，行动越多；仪式感越强，留存越好。
> - 单任务模式：一次只看一个任务，减少选择焦虑
> - 完成庆祝：30 秒的「成就时刻」，强化正向循环
> - 慢慢来，比较快

---

### 马斯克原则（第一性原理视角）

**原则 4：第一性原理思考**
> 为什么任务管理一定要有长长的任务列表？
> - 因为以前的软件都是这么做的？
> - 还是因为用户真的需要？
>
> 真相：用户打开任务管理是为了「减少焦虑」和「知道现在该做什么」，不是为了「看一个长长的列表」。

**原则 5：顺着人性，不要对抗人性**
> 拖延 = 大脑的节能机制，不是道德缺陷。
> - 坏的产品：「你怎么还不做？」
> - 好的产品：「我理解，不想开始是很正常的。要不试试先做 5 分钟？」

**原则 6：反常识创新**
> 所有人都在做「更多」，我们做「更少」；
> 所有人都在「推」用户，我们「保护」用户。

### 4.1 核心区别对比

**AI 学习到的用户模型：
```typescript
interface UserEnergyModel {
  // 星期几的能量模式：周一低，周三高，周五最低
  weekdayPattern: number[]
  
  // 一天中不同时间段的能量
  hourlyPattern: number[]
  
  // 不同任务类型的偏好
  taskTypePreference: Record<string, number>
  
  // 最近 7 天的平均能量
  recentEnergy: number
}
```

### 8.2 推荐算法

```
推荐分数 = 优先级权重 × 时间邻近度 × 能量匹配度 × 情绪匹配度

其中：
- 优先级权重：P0=5, P1=4, P2=3, P3=2, P4=1, P5=0.5
- 时间邻近度：今天=1.5, 明天=1.2, 以后=1.0, 逾期=2.0
- 能量匹配度：任务能量需求 ≈ 用户当前能量 → 1.5，否则递减
- 情绪匹配度：用户最近完成类似任务的情绪标签匹配 → +0.2
```

## ============================================================
## 九、反目标保护模式
## ============================================================

### 核心洞察

> **问题**：OKR 模式的致命缺陷 → 目标 = 压力 = 焦虑
>
> **解决方案**：反目标模式 → 不设目标，只记录「我今天做了什么」

### 反目标模式的三个核心：

1. **没有「截止日期」，只有「创建日期」
2. **没有「必须完成」，只有「正在进行」**
3. **没有「目标进度条**，只有「完成记录」**
4. **没有「逾期」**，只有「还在做」**

### 适用场景

- 创意工作者、创业者、自由职业者
- 不想被目标推着走的人
- 已经被 OKR 搞得焦头烂额的人
