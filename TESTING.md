# Merize 功能测试文档

本文档列出本次新增功能的完整测试步骤和验收条件，请按照顺序逐一测试。

---

## 前置条件

确保你已经在项目根目录安装了依赖：

```bash
npm install
```

桌面端真实验证请优先使用：

```bash
npm run tauri dev
```

只有在做纯前端 demo 或样式验证时，才使用：

```bash
npm run dev
```

说明：

- `npm run tauri dev` 才会走真实的桌面 IPC、追踪状态和本地持久化路径。
- `npm run dev` 只是 web demo，不代表打包后的桌面应用真实行为。
- 任何“后台自动追踪”“退出重开后数据仍在”“打包后可用”的验收，都必须在桌面模式下完成。

## 测试列表

### 测试 1：语言切换功能 (i18n)

**测试路径：** 进入 `Settings` (设置) 页面

**测试步骤：**

1. 在设置页面顶部找到 "Language / 语言" 区域
2. 点击 "English" 按钮
   - **预期结果：** 整个界面立刻变成英文
   - **验收条件：** 所有文字都是英文，包括导航栏、各个区域标题、按钮文字
3. 点击 "简体中文" 按钮
   - **预期结果：** 整个界面立刻变回中文
   - **验收条件：** 所有文字都是中文
4. 刷新浏览器页面
   - **预期结果：** 保持你上次选择的语言
   - **验收条件：** 语言选择持久化，不会变回默认

**验收标准：**
- [ ] 切换即时生效，不需要刷新
- [ ] 所有界面文字都正确翻译，没有漏翻译的键名（比如不应该出现 `settings.aiApiKey` 这种原文）
- [ ] 刷新后保持语言选择

### 测试 2：AI 设置 - API Key 保存和多提供商选择

**测试路径：** 进入 `Settings` → 找到 "AI Classification Settings / AI 分类设置" 区域

**测试步骤：**

1. 检查提供商列表
   - **预期结果：** 应该看到 **9 个**提供商选项：
     - 文心一言 (百度)

     - 智谱清言 (智谱)

     - Claude (Anthropic)
     - Gemini (Google)
     - DeepSeek
     - XAI
   - **验收条件：** 9 个全部显示，每个都有中文名称和英文描述

2. 选择不同的提供商
   - **预期结果：** 选中的提供商有高亮边框和对勾标记
   - **验收条件：** 点击哪个哪个高亮，只有一个选中

   - **预期结果：** 输入框是密码类型，输入的内容被隐藏（只显示圆点）

   - **预期结果：** 保存成功应该有 Toast 提示成功
   - **验收条件：** 设置成功保存，刷新页面后 API Key 和提供商选择保持不变

**验收标准：**
- [ ] 所有 9 个提供商都显示正确

**验收标准：**
- [ ] 点击按钮能正确下载

### 测试 4：仪表盘拖拽自定义排序

**测试路径：** 进入 `Dashboard` (仪表盘) 页面

**测试步骤：**

1. 观察仪表盘组件
   - **预期结果：** 默认顺序应该是：
     1. 顶部问候横幅 (Tracking Banner)
     2. 快捷操作 (Quick Actions)
     3. 统计卡片行 (Stats Row)
     4. 计划对比 (Plan Comparison)
     5. 主时间线 (Main Timeline)
     6. 侧边小部件 (Sidebar Widgets)
     7. 分类饼图 (Category Breakdown)

2. 尝试拖拽
   - **操作方法：** 鼠标放在任意组件上，看右上角会出现 `⋮⋮` 拖拽手柄，按住拖拽
   - **预期结果：** 拖拽时原位置空出，拖拽的组件半透明跟随鼠标
   - **验收条件：** 拖拽流畅，放下后组件顺序正确更新

3. 把最下面的 "分类饼图" 拖到最上面
   - **预期结果：** 刷新页面后，分类饼图仍然在最上面
   - **验收条件：** 排序结果持久化保存

4. 尝试不同的排列组合
   - **预期结果：** 任意顺序都能正确保存，显示不乱

**验收标准：**
- [ ] 每个组件右上角悬浮显示拖拽手柄
- [ ] 拖拽过程有正确的视觉反馈（半透明、阴影）
- [ ] 放下后顺序正确更新
- [ ] 刷新浏览器，顺序保持不变，不会恢复默认

### 测试 5：Timeline 活动粒度滑块

**测试路径：** 进入 `Timeline` (时间线) 页面

**测试步骤：**

1. 找到 "Granularity / 粒度" 滑块（在页面顶部筛选区域）
   - **预期结果：** 滑块默认在中间位置 (Level 3)
   - **描述应该是：** "Balanced - 平衡"

2. 把滑块拖到最左边 (Level 1)
   - **预期结果：** 描述变成 "Coarsest - 最粗"
   - **验收条件：** 时间线上只显示合并后的大时间段，很多短活动被合并了

3. 把滑块拖到最右边 (Level 5)
   - **预期结果：** 描述变成 "Finest - 最细"
   - **验收条件：** 时间线上显示所有活动，哪怕只有几分钟的也单独显示

4. 尝试不同位置，观察时间线活动数量变化
   - **预期结果：** 滑块越往左 → 活动越少（合并越多），滑块越往右 → 活动越多（显示越细）

**验收标准：**
- [ ] 滑块拖动即时更新时间线
- [ ] 粒度标签和描述正确显示当前级别
- [ ] 不同粒度下活动数量变化符合预期（越细越多，越粗越少）
