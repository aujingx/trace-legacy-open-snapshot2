> 最后更新：2026-04-28
> 状态：待讨论确认

## 🎯 核心定位

用户打开应用看到的第一个页面。

**目标**：帮助用户用最低的门槛开始一天的工作，客观展示进度，温和引导。

**设计原则**：
- ✅ **启动优先**：最显眼的位置是让用户「开始做事」的按钮，不是让用户「看数据」
- ✅ **客观不评判**：只说「你今天专注了 2 小时 13 分」，不说「你比昨天少了 50 分钟」
- ✅ **温和不打扰**：建议永远是可选的，永远可以忽略
- ✅ **信息密度适中**：不要太空，也不要堆太多东西让人焦虑

## 📐 每个区域的详细设计

### 1. 全局 AI 助手入口

**设计原则**：**一处添加，多页可见**，不绑定在单一页面

#### 入口方式（三种并行）：

1. **全局快捷键唤起**（类似 Spotlight/Alfred）
   - `Cmd/Ctrl + K` 随时呼出输入框，悬浮在任意页面之上
   - 操作完成自动消失，结果同步更新到当前页面
   - 比如在 Timeline 页面添加任务 → Timeline 上直接出现新任务条目
   - 比如在 Task 页面改时间 → 任务列表里直接更新时间

2. **Dashboard 常驻输入框**
   - 位置：顶部或底部（P1 支持用户选择）
   - 样式和快捷键唤起的保持一致
   - 操作结果直接更新当前页面的任务列表

3. **各页面的快速添加按钮**
   - Task 页面的「+ 添加任务」按钮
   - Timeline 页面的「+」按钮
   - 点击按钮后先弹出轻量选择器：「📝 表单添加」/「🤖 AI 添加」
   - 选择「AI 添加」→ 按钮附近弹出小输入卡片（和快捷键唤起的内容一样，只是位置就近）
   - 输入和确认操作都在卡片内完成，操作成功后卡片自动收起，直接更新当前页面内容
   - 全程不跳页、不弹大窗口

#### 统一交互逻辑：
- 输入文字按回车 → AI 展示理解的结果卡片
- **必须用户明确点击「确认」才执行**，没有自动确认
- 支持短对话追问：信息不明确时 AI 会简单问「这个任务大概需要多久呢？」
- 操作成功后**直接更新当前页面的内容**，不弹 Toast
- 撤销：所有 AI 操作记录在「操作历史」抽屉（右下角小图标），随时可以点进去撤销
- 桌面应用追求安静，避免不必要的弹窗和 Toast 打断用户

---

### 2. 任务列表区

**位置**：输入框下方

**标题**：「🌱 从这里开始吧」（温和引导，不强调完整性或推荐）

**规则**：
- 最多展示 5 个任务
- P0 按优先级排序，P1 再做智能时间匹配排序
- 超过 5 个的话，底部放「查看全部任务」按钮

**每个任务卡片显示**：
- 🔴🟡🟢 优先级色标（左边缘）
- [ ] 复选框（点击直接标记完成）
- 任务标题
- ⏱️ 预估时间
- 最右侧 ▶️ 小图标按钮（点击直接开始专注）
- 📌 一行小字的智能上下文提示（P1 实现，比如「建议上午做（你上午效率最高）」）

**设计考量（轻量化方案 B）**：
- 每个任务自带启动入口，用户不需要先选任务再去顶部点启动
- 用 ▶️ 小图标代替大按钮，视觉上更清爽，不抢任务标题的注意力
- 鼠标 hover 时图标微微放大 + 显示 tooltip「开始专注」
- 点击后直接进入专注模式，时长优先用任务预估时间，没有预估则默认 25 分钟
- 不区分「开始专注」和「开始 5 分钟」，统一入口，进入专注模式后用户可随时调整时长

**智能排序逻辑（P1 实现）**：
```
推荐分数 = 
  优先级权重 
  + 截止日期临近权重 
  + 时间段匹配度（难的放上午，简单的放下午）
  - 被推迟次数的惩罚分
  + 上一个任务完成后的休息缓冲逻辑
```

### 3. 今日概览卡片

**位置**：任务列表下方

**展示 3 行关键信息，不带任何评判色彩**：

```
⏱️ 已专注 2h13m  /  计划 4h
✅ 已完成 2 个  /  共 4 个
🔥 连续第 3 天在 10 点前开始工作
```

**设计原则**：
- 只说事实，不说「好」或「不好」
- 「计划 4h」是用户自己设的目标，不是系统给的
- streak 是正反馈，不是考核（连续 N 天只是陈述事实）
- 没有进度条的红色/绿色，就是纯文字，避免给人压力

---

### 4. 智能提示区

**位置**：页面最底部，Dashboard 专属的轻量提示

**设计原则**：
- **纯展示，无交互**：不需要用户选择/点击确认，只是温和提醒
- **不需要用户回应**：去掉「好的/不用/忽略」按钮，减少决策负担
- **有就显示，没有就消失**：最多同时展示 1 条
- **重要提示走系统通知**：考虑到用户可能主要使用 Timeline 不常来 Dashboard，真正重要的提醒同时通过系统通知推送

**提示类型（非任务建议，专注效率和行为优化）**：
- 专注太久了：「你已经连续专注 90 分钟了，站起来走 2 分钟吧」
- 效率模式：「你今天上午效率很高！可以乘胜追击把最难的做了」
- 分心提醒：「今天切换应用有点频繁，试试开专注模式屏蔽一下」
- 习惯引导：「你这一周都是下午 2 点开始工作，试试把重要任务安排在这个时间段」

## 🧩 全局通用组件

### 1. 操作历史抽屉（右下角常驻小图标）

**入口**：所有页面右下角都有一个 🕒 小图标，点击从右侧滑出抽屉

**记录内容**：
- AI 操作记录：添加任务、修改时间、重排日程、拆分任务等
- 手动操作记录：手动创建/修改/完成任务
- 活动追踪记录：自动检测到的专注、分心等活动
- 每条记录显示：时间、操作类型、简要内容

**交互**：
- 所有可撤销的操作右侧都有「撤销」小字链接
- AI 操作可以点击展开看当时的完整对话
- 点击活动记录可以跳转到 Timeline 对应时间点
- 支持按天筛选查看历史

**设计原则（保持轻量）**：
- 不主动弹窗，用户想看才点开，平时就是一个不起眼的小图标
- 列表展示，只显示关键信息，不做复杂的可视化
- 只保留最近 7 天的记录，超过自动清理，不臃肿
- 不后台同步、不做复杂计算，就是本地简单的日志记录

## 🧩 Dashboard 模块化说明

Dashboard 所有区域均为独立可插拔组件：
1. 🎙️ 自然语言输入框组件
2. 📋 任务列表组件
3. 📊 今日概览卡片组件
4. 💡 智能提示区组件

**未来支持**：
- 用户自定义 Dashboard：添加/移除组件、调整顺序
- 每个组件可独立设置默认显示/隐藏状态
- 每个组件可独立更新，不影响其他区域

## 🎯 P0 阶段的简化版

P0 我们先做最简单能用的版本，砍掉智能的部分：

### P0 Dashboard 只做：
1. ✅ 顶部自然语言输入框（完整的 P0 功能）
2. ✅ 任务列表（按优先级排序，最多 5 个），每个任务右侧带「开始专注」按钮
3. ✅ 今日概览卡片（只显示专注时长和完成任务数，没有 streak，没有智能提示）

### P0 暂时不做的：
- ❌ 任务的智能排序逻辑（先简单按优先级排）
- ❌ 每个任务下面的小字智能上下文提示
- ❌ 智能提示区（默认折叠的效率提示）
- ❌ streak 统计
- ❌ Dashboard 自定义模块化功能

## 🔗 和其他模块的关系

| 模块 | 交互 |
|------|------|
| **自然语言 Agent** | Dashboard 顶部的输入框是它的唯一入口 |
| **专注模式** | 点击启动按钮直接进入 |
| **任务管理** | 推荐任务列表来自任务系统，点击复选框直接改状态 |
| **事件总线** | 所有操作（开始专注、完成任务）都发布对应事件 |
| **正反馈系统** | P1 接入后，达成 streak 时在概览卡片展示 |
| **规则引擎** | P1 接入后，生成的建议展示在底部建议区 |

## 🎨 设计语言统一

- 所有卡片：12px 圆角，轻微阴影，白色背景
- 按钮：8px 圆角，主按钮蓝色 #3B82F6，次按钮浅黄 #FEF3C7
- 字体：正文 14px，标题 16px，数字 20px
- 间距：卡片之间 16px，内容和卡片边缘 20px padding
- 颜色：尽量柔和，不要太刺眼的饱和色

## 📋 待讨论确认的问题

1. **两个启动按钮的文案**：「开始专注 25 分钟」 + 「先做 5 分钟试试」，这个文案 OK 吗？
2. **任务推荐区最多 3 个**：会不会太少？还是最多 5 个？
3. **今日概览卡片的内容**：现在的 3 行信息够不够？有没有多余或者缺的？
4. **P0 简化范围**：P0 先不做智能排序和建议，你觉得可以吗？
5. **整体信息密度**：你觉得现在这个布局是太挤了，还是太空了？
